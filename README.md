# 🎭 TrollDrop

TrollDrop is a CS2 plugin that secretly rigs dropped weapons with hidden traps. When a player drops a weapon, the server silently rolls **one** of three troll effects and assigns it to that weapon — the enemy never knows what's waiting for them!

## ✨ Features

- 3 distinct troll effects: **Bounce**, **Spin**, and **Swap**
- Only **one** effect is rolled per drop — fully random and hidden from players
- The dropper is immune: the zone only activates after they leave the trigger radius
- Configurable weighted pool — disable any effect by setting its weight to `0`
- Automatic zone expiration — zones clean themselves up after a configurable lifetime
- Reload config at runtime without restarting the server

## ⚙️ Installation

1. Download the plugin release.
2. Copy the entire `TrollDrop` folder (with all its contents) to your CS2 server's `plugins` directory (`addons/counterstrikesharp/plugins`).
3. The configuration file will be automatically generated by the plugin after the first run.
4. Restart your server.

## 🎮 How it works

When a weapon is dropped on the ground, the server secretly rolls **one** troll effect and assigns it invisibly to that weapon:

- **Bounce** — The weapon flies away from anyone who gets too close. The closer the player, the harder the kick.
- **Spin** — The approaching player's camera snaps 180° instantly and they get launched forward — completely disorienting.
- **Swap** — Nothing visible happens on approach. But the moment the player picks the weapon up, it's silently replaced with a random weapon from the configured pool. ⚠️ *This effect has not been fully tested — use at your own risk and report any issues!*

> The dropper is never affected by their own trap. The zone stays dormant until they physically leave the trigger radius.

## 🛠️ Configuration

The configuration file is automatically generated after the first run at:
`addons/counterstrikesharp/configs/plugins/TrollDrop/Config.toml`

```toml
enabled = true

min_bounce_force = 400.0
max_bounce_force = 700.0
trigger_radius = 220.0
arm_delay = 1.5
zone_lifetime = 30.0
bounce_cooldown = 0.3
check_interval = 0.05
spin_forward_speed = 500.0
spin_launch_force = 300.0

[proximity_events]
bounce = 34
spin   = 33
swap   = 33

[swap_weapons]
weapon_negev  = 50
weapon_xm1014 = 50
```

### General settings

- `enabled`: Master switch. Set to `false` to disable the plugin without unloading it.
- `min_bounce_force`: Upward velocity applied to the weapon when a player is at the edge of the trigger radius (softer bounce).
- `max_bounce_force`: Upward velocity applied when a player is nearly touching the weapon (hardest bounce).
- `trigger_radius`: Radius in Hammer units around the dropped weapon that activates its effect. CS2 reference: ~65 units ≈ 1 meter. Default `220` ≈ ~3.4 meters.
- `arm_delay`: Seconds after a weapon is dropped before the zone becomes active.
- `zone_lifetime`: How long (in seconds) a zone stays alive before automatically disappearing.
- `bounce_cooldown`: Minimum time (seconds) between consecutive triggers of the same zone. Prevents effects from firing every tick.
- `check_interval`: How often (seconds) the proximity check runs. Lower = more responsive. `0.05` = 20 checks per second.
- `spin_forward_speed`: Horizontal speed (units/s) the player is launched at when the Spin effect fires.
- `spin_launch_force`: Upward velocity added on Spin — gives it a jump feel. Set to `0` for a pure horizontal shove.

### `[proximity_events]` — effect weight pool

Controls which effect gets rolled when a weapon is dropped. Weights are relative and do **not** need to sum to 100. Set a weight to `0` to disable that effect entirely.

- `bounce`: Weight for the Bounce effect.
- `spin`: Weight for the Spin effect.
- `swap`: Weight for the Swap effect.

### `[swap_weapons]` — swap replacement pool

The weapon pool used when the Swap effect fires on pickup. Format: `weapon_designer_name = relative_weight`.

When triggered, the plugin checks what the player had in their **active hand** at the moment of pickup and replaces only that weapon. Knives are **never** removed or replaced.
